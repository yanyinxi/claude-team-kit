description: 每次分析应激活专家模式多轮深度思考，循环检查直到边际收益为零，不应每次被追问才发现新问题
type: feedback
---

分析代码时必须在第一轮就穷尽所有问题，不能每次被追问才深入一层。

**Why:** 用户观察到每次分析都能发现新问题（第1轮22个→第2轮3个→第3轮80个），说明前几轮思考深度不足，没有激活专家模式的强制深度思考。

**How to apply:**
- 代码分析任务必须激活专家模式 + 强制深度思考
- 第一轮就要并行派发所有专业 Agent 做逐文件穷举审查
- 不能停留在"发现几个代表性问题就停"的层次
- 审查标准：直到发现的问题开始重复或降为纯风格偏好才停止
- 每一轮自我追问："还有没有我没看到的文件/模块/层次？"
