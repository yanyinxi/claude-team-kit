# 需求分析技能

## 何时使用

当需要以下操作时使用此技能：
- 分析用户原始需求
- 生成产品需求文档（PRD）
- 拆解用户故事（User Stories）
- 定义验收标准
- 评估任务优先级

## 工作流程

### 第一步：理解需求
- 仔细阅读用户需求
- 识别核心问题和用户目标
- 必要时提出澄清问题

### 第二步：分析需求
- 提取功能需求
- 识别非功能需求（性能、安全、UX）
- 评估技术可行性
- 考虑边界情况

### 第三步：创建 PRD
生成包含以下内容的 PRD 文档：

```markdown
# 产品需求文档：[功能名称]

## 1. 概述
- 问题陈述
- 解决方案描述
- 成功指标

## 2. 用户故事
作为 [用户类型]，我想 [操作]，以便 [好处]。

## 3. 功能需求
- 需求 1
- 需求 2

## 4. 非功能需求
- 性能：[具体指标]
- 安全：[安全要求]

## 5. 验收标准
- [ ] 标准 1
- [ ] 标准 2

## 6. 技术考虑
- API 端点需求
- 数据库变更
- 第三方集成
```

### 第四步：任务拆分和优先级
- 将复杂功能拆分为可执行的开发任务
- 基于业务价值和技术复杂度评估优先级
- 识别任务依赖关系

## 最佳实践

1. **具体清晰**：避免模糊的需求描述
2. **可测试**：每项需求都应该是可测试的
3. **以用户为中心**：关注用户价值
4. **可测量**：验收标准应该可测量、可验证
