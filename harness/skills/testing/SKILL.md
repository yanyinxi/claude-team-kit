# 测试技能

## 何时使用

当需要以下操作时使用此技能：
- 设计测试用例
- 编写单元测试
- 执行集成测试
- 生成测试报告

## 工作流程

### 第一步：分析需求
- 阅读验收标准
- 识别测试场景
- 列出测试点

### 第二步：设计测试用例
- 正向测试（Happy Path）
- 负向测试（边界情况）
- 异常处理测试

### 第三步：编写测试代码
- 使用合适的测试框架
- 遵循测试金字塔原则
- 保持测试独立

### 第四步：执行测试
- 运行单元测试
- 运行集成测试
- 记录测试结果

### 第五步：生成报告
- 汇总测试结果
- 记录失败的测试
- 提供修复建议

## 输出格式

```markdown
# 测试报告：[功能名称]

## 测试概览
- 总测试数：XX
- 通过：XX
- 失败：XX
- 通过率：XX%

## 测试用例

### 功能测试
| 用例 ID | 描述 | 预期结果 | 实际结果 | 状态 |
|--------|------|----------|----------|------|
| TC001 | 测试登录成功 | 返回 token | 通过 | ✅ |
| TC002 | 测试密码错误 | 返回错误 | 通过 | ✅ |

### 边界测试
| 用例 ID | 描述 | 预期结果 | 实际结果 | 状态 |
|--------|------|----------|----------|------|
| TC101 | 空用户名 | 提示必填 | 通过 | ✅ |
```

## 最佳实践

1. **测试金字塔**：大量单元测试，适量集成测试，少量 E2E 测试
2. **独立性**：每个测试独立运行
3. **可重复**：测试在任何环境可重复执行
4. **快速反馈**：测试应该快速执行
5. **测试位置**：所有测试必须放在 main/tests/ 目录下

## 测试目录结构

```
main/tests/
├── backend/              # 后端测试
│   ├── test_auth.py
│   ├── test_api.py
│   └── test_services.py
├── frontend/             # 前端测试
│   ├── components.test.ts
│   └── services.spec.ts
├── integration/          # 集成测试
│   └── test_e2e.py
└── conftest.py           # Pytest 配置
```
