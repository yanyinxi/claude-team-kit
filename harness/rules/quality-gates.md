# Quality Gates — 每个阶段的退出条件

## 需求阶段

- [ ] PRD 包含目标、用户故事、功能需求、非功能需求、验收标准
- [ ] 所有假设已明确并得到确认
- [ ] 验收标准具体且可测试

## 设计阶段

- [ ] 有方案对比（≥2 个方案）
- [ ] 有 Trade-off 分析
- [ ] 有风险评估和缓解方案
- [ ] 影响范围已识别

## 实现阶段

- [ ] 代码通过 lint 和类型检查
- [ ] 所有测试通过
- [ ] 新增代码有对应测试
- [ ] 无硬编码 Secret 或 Token

## 审查阶段

- [ ] 至少一个审查 Agent 通过
- [ ] 安全相关代码经过 security-auditor
- [ ] 审查意见已处理或明确驳回

## 交付阶段

- [ ] 所有测试通过（含回归）
- [ ] 构建成功
- [ ] 关键性能指标未退化
- [ ] 文档已更新（如需要）
