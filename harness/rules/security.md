# Security Baseline — 安全底线

## 绝对不做的事

- **生产环境数据**：不使用真实用户数据做测试
- **硬编码 Secret**：API Key、密码、Token 永远不过代码提交
- **字符串拼接 SQL**：所有数据库查询必须参数化
- **裸抛异常**：错误响应不泄露内部实现细节
- **跳过认证**：不在测试环境关闭认证逻辑

## 必须做的事

- **输入验证**：所有外部输入经过验证和清理
- **最小权限**：数据库连接、API 调用使用最小必要权限
- **日志脱敏**：密码、Token、身份证号、手机号不入日志
- **依赖审计**：定期检查依赖是否有已知漏洞
- **HTTPS**：生产环境强制 HTTPS

## 安全审查触发条件

以下情况必须经过 security-auditor：
- 认证/授权逻辑变更
- 数据库查询方式变更
- 用户输入处理逻辑变更
- 第三方 API 集成
- 文件上传/下载功能
