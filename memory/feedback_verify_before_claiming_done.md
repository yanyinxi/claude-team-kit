任何"已完成"的声明都必须有证据支持。没有证据 = 没有完成。

**Why:** AI 在工作中多次出现虚假声明：声称创建了文件但文件不存在，声称发现了问题但没有具体证据。这导致用户无法信任 AI 的报告。

**How to apply:**
- **创建文件后**：必须用 `ls -la <path>` 或 `Read` 工具验证文件存在
- **发现问题后**：必须记录具体问题和发现位置（文件名:行号）
- **修改代码后**：必须用 `git diff` 或 `Read` 验证修改内容
- **运行测试后**：必须记录测试输出

**禁止的行为：**
- ❌ 在 todo 中记录"已完成"但实际未完成
- ❌ 声称创建了文件但没有验证文件存在
- ❌ 声称"发现了 N 个问题"但没有具体列表

**验证清单：**

| 声明类型 | 验证方式 |
|----------|----------|
| 创建了文件 | `ls -la <path>` 或 `Read` 工具 |
| 发现了问题 | 记录具体问题和发现位置 |
| 修改了代码 | `git diff` 或 `Read` 验证修改内容 |
| 运行了测试 | 记录测试输出 |

**决策口诀：** "我声称完成了什么？证据在哪？→ 没有证据就不能声明完成。"
